============================================================
            本地知识库问答系统 · 使用说明书
============================================================

这是一个完全在本机离线运行的中文知识库问答程序。把你的 PDF/Word
资料放进去,它会建立一个“知识库”,之后你用大白话提问,它会从你的
资料里找答案并作答。全程不联网、不上传,资料只留在这台电脑上。

无需安装 Python、无需安装 Ollama,双击即可使用。


------------------------------------------------------------
一、你看到的文件夹长什么样
------------------------------------------------------------

知识库问答\
├─ config.yaml          ← 配置文件(进阶可改,见第六节)
├─ models\              ← 模型文件(已放好,请勿删除)
├─ docs_kb\             ← 【你把要问的 PDF/Word 放这里】
├─ chroma_db\           ← 知识库数据(建库后自动生成)
├─ ① 建库.bat          ← 第一步:建立/更新知识库
├─ ② 启动.bat          ← 第二步:启动问答
├─ 使用说明.txt         ← 本文件
└─ bin\                 ← 程序本体(升级时只替换这里,数据不受影响)
    ├─ doctor-t.exe     ← 主程序(别直接双击,用上面的 .bat)
    └─ _internal\       ← 程序运行所需文件,【请勿删除或改动】

整个文件夹可整体拷贝/移动。注意:程序在 bin\ 子文件夹里,数据(config/models/
chroma_db/docs_kb)在外层——这样升级程序(替换 bin\)绝不会动到你的知识库。


------------------------------------------------------------
二、第一次使用(三步)
------------------------------------------------------------

第 1 步:放资料
    把要查询的 PDF 或 Word 文档复制到 docs_kb\ 文件夹里。
    （可以放多个文件,支持中文。)

第 2 步:建立知识库
    双击「① 建库.bat」。
    会弹出一个黑色窗口,显示“解析文档 → 切分 → 编码 → 写入”等进度。
    文档越多越大,耗时越久(纯 CPU,可能几分钟到几十分钟)。
    看到“重建完成:N 个文档,M 个片段”即成功。按任意键关闭窗口。

第 3 步:启动问答
    双击「② 启动.bat」。
    稍等片刻(首次加载模型较慢,可能 10~60 秒),浏览器会自动打开
    问答页面。在输入框里提问、回车,即可得到基于你资料的回答。

    想结束:关闭那个黑色命令行窗口即可。


------------------------------------------------------------
三、日常使用
------------------------------------------------------------

· 以后每次用,只需双击「② 启动.bat」。
· 浏览器没自动弹出?手动打开浏览器,访问:  http://127.0.0.1:7860
· 回答下方可展开“🔍 召回片段”,查看答案是依据资料里的哪几段得出的。
· 资料有增减/更换后,需要重新双击「① 建库.bat」让知识库更新,
  否则问答用的还是旧资料。


------------------------------------------------------------
四、给其他设备访问(同一局域网,选用)
------------------------------------------------------------

程序默认也对局域网开放。同一 WiFi/网线下的其他电脑或手机,可在浏览器
访问:  http://本机IP:7860
查看本机 IP:在命令行运行 ipconfig,看“IPv4 地址”。
如访问不了,检查 Windows 防火墙是否放行 7860 端口。
(另有一个程序接口在 8001 端口,供开发者调用,普通用户无需关心。)


------------------------------------------------------------
五、常见问题
------------------------------------------------------------

Q1. 双击后窗口一闪就没了 / 启动失败?
    用「② 启动.bat」启动(它会在出错后保留窗口并显示原因),
    或在文件夹空白处按住 Shift 右键→“在此处打开 PowerShell”,输入:
        .\知识库问答.exe --mode both
    把窗口里红色的报错信息记下来反馈。最常见原因见 Q2。

Q2. 提示找不到模型 / config 里 device 之类的错误?
    打开 config.yaml(记事本即可),确认:
      - embedding.device 为 cpu
      - retrieval.reranker_device 为 cpu
      - llm.model 指向 models 文件夹里实际的 .gguf 文件名
    且 models\ 里这几样都在:生成模型(.gguf)、bge 向量模型文件夹、
    重排模型文件夹。改完保存,重新建库/启动。

Q3. 回答很慢?
    纯 CPU 运行,生成速度本就有限(尤其首次要加载模型)。
    文档多时建库也慢,属正常。耐心等待,或减少文档量。

Q4. 回答“知识库中没有相关内容”?
    说明你的资料里确实没有相关信息,或还没建库/资料没放进 docs_kb。
    确认已双击过「① 建库.bat」且文档在 docs_kb 里。

Q5. 回答不准 / 答非所问?
    本程序只依据你提供的资料作答,不会凭空编造。可换种问法,或在
    “🔍 召回片段”里核对它找到的原文段落。

Q6. 端口被占用 / 打不开网页?
    可能 7860 端口被别的程序占用。可在 config.yaml 的 server 段
    改 web_port 为其他端口(如 7861),保存后重启。


------------------------------------------------------------
六、配置文件 config.yaml(进阶,可不改)
------------------------------------------------------------

用记事本打开,常用项:

  llm.model            生成模型文件名(放在 models\ 下)。想换更强/更快的
                       模型,把对应 .gguf 放进 models\ 并改这里的文件名即可。
  llm.max_tokens       回答最大长度,觉得回答被截断可调大。
  embedding.device     必须为 cpu(本机无独立显卡时)。
  retrieval.reranker_device  必须为 cpu。
  retrieval.top_k      每次喂给模型的资料片段数。
  server.web_port      问答网页端口(默认 7860)。

修改后保存,重新双击「② 启动.bat」生效。
改动 llm/embedding 相关后,建议重新建库一次更稳妥。


------------------------------------------------------------
七、注意事项
------------------------------------------------------------

· 全程离线,资料不出本机,适合敏感/内部资料。
· _internal\ 与 models\ 是程序运行核心,请勿删除或改名。
· 程序占用内存较高(要加载多个模型),建议内存 ≥ 8GB。
· 关闭程序:直接关掉那个黑色命令行窗口。
· 整个“知识库问答”文件夹可整体备份/迁移。
